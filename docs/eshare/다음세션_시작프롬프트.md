### 작업 내용
1. **스캐폴딩** — 신규 프로젝트 생성(기존 kakao_mcp와 완전 분리)

   - 배움알리미 `server.py` 골격·`Dockerfile`(빌드타임 SQLite COPY)·`requirements.txt`(mcp==1.26.0) 복제 후 eShare용으로 수정. stateless_http=True, streamable-http.

2. **SQLite 스키마 + 목업 DB** — `API_스펙_정리.md` 필드맵 그대로 DDL 작성 (`resources`, `agencies`, `resource_classes` 테이블). 목업 데이터 50건+(지역·분류 다양성, 실제 분류코드 010100/020000 등 사용) 생성 스크립트 `scripts/build_mock_db.py`.

3. **어댑터 + 적재 파이프라인** — `adapters/eshare_api.py`(POST+JSON 클라이언트, resultCode 검사, 401 시 "승인 대기" 안내 에러) + `scripts/build_db.py`(분류별 목록 순회 → rsrcNo 수집 → 상세 100건 배치 → SQLite 적재, updYmd 증분 지원, 스냅샷 타임스탬프 기록). 인증키는 환경변수 `ESHARE_API_KEY`.

4. **MCP tool 4종** (전부 read-only annotations 5종, description 영문+"공유누리/eShare" 병기, 이름에 kakao 금지)
   - `search_resources`: region(시구군)/resource_type(분류코드)/keyword(inqTag·rsrcNm)/free_only/page 다기준 검색
   - `get_resource_detail`: rsrcNo → 상세 + **예약채널(instUrlAddr, dtlUrlAddr) 필수 노출**
   - `recommend_resources`: purpose/region/free 기반 **고정 가중합 결정론 랭킹**(area·요금·분류 매칭, volatile 배제, 동일입력=동일출력)
   - `get_filter_options`: 분류·지역 enum + **교차기관 집계/분포 통계**(결정론 커널)
   - 전 tool 응답에 footer 자동 부착: 스냅샷 기준시점 + "실시간 예약가능 여부와 다를 수 있음, 예약은 공유누리에서" 디스클레이머. finalize()로 24KB 가드.

5. **테스트** — `tests/`에 수용기준 assert: ①4 tool 응답 24KB 이내 ②전 응답에 타임스탬프+디스클레이머, 실시간 단언 문구 부재 ③결정론 커널 산출 검증 ④recommend 동일입력 재현 ⑤annotations 5종·Resource/Prompt 미노출 ⑥성능(로컬 평균<100ms). 목업 DB로 전부 그린.

6. **심사 대비 문서** — `docs/반려_셀프체크.md`(배움알리미 것 참조·복제), tool description 최종본, README.

7. **검증** — MCP Inspector 또는 streamable-http 클라이언트로 E2E 확인 후 완료 보고.
