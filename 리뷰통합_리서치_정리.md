# PART 1. 위험성 조사

## 1-1. 시장 리서치 — 경쟁/선례

**핵심: 정확히 이 형태(소비자용 멀티플랫폼 리뷰 통합)는 한국에 명확한 강자가 없는 빈 영역. 단 "빈 시장"이 아니라 "아무도 깔끔하게 못 푼 어려운 시장".**

| 영역 | 사례 | 우리와 차이 |
|---|---|---|
| 🎯 정중앙(목표) | 거의 없음 | — |
| 해외 유사 | SeekShop(가장 근접), FakeFind, Shopfox | 미국 리테일러 기반, 한국몰 ❌ |
| 죽은 선구자 | **Fakespot**(2025.7 종료), **ReviewMeta**(2026초 종료) | 아마존 단일·가짜리뷰 "판별"만 |
| 한국 B2B(레드오션) | 크리마·브이리뷰·알파리뷰·리글 | 판매자가 자기몰로 리뷰 모으는 솔루션 |
| 한국 가격비교 | 다나와·네이버쇼핑 | 가격은 통합, 리뷰는 자사 생태계만 |
| 가격추적 확장 | 폴센트·쿠스피·내려가개 | **가격만**, 리뷰 전무 |
| 플랫폼 내장 AI요약 | 네이버·SSG·배민·아마존 | 자기 플랫폼 리뷰만 |

- **수요 확인** ✅ — "광고·가짜 리뷰 많아 진짜 후기 찾기 어렵다", "여러 곳 비교되면 좋겠다"는 호소 다수
- **경고 신호** ⚠️ — 최강자 Fakespot(1000만 유저)도 ① 플랫폼 적대(아마존이 앱 퇴출) ② 무료 수익화 실패 ③ 데이터 접근 제약으로 **사망**

## 1-2. 법적 결론 — 크롤링은 왜 위험한가

**"비영리면 괜찮다"는 안전하지 않음.** 합법성을 가르는 건 영리 여부가 아니라 5개 축:

| 축 | 영리 무관하게 문제? |
|---|---|
| ① 로그인·캡챠 우회 수집 | ✅ 형사(정보통신망법) |
| ② DB 상당부분 복제·축적 | ✅ 저작권법 DB권 |
| ③ 서버 과부하 | ✅ 컴퓨터등업무방해 |
| ④ 경쟁 영업에 전용 | ⚠️ 영리·경쟁일 때(부정경쟁) |
| ⑤ 원문 재게시(공중 제공) | ⚠️ 비영리여도 위험 |

**판례**: 잡코리아 vs 사람인 → DB권 침해 **4.5억 배상** / 야놀자 vs 여기어때 → **공개 데이터였는데도** 부정경쟁 민사 **10억 배상**(형사는 무죄)

**→ "비로그인 공개 데이터만"으로 한정하면 형사(①③)는 거의 해소되지만, "원문을 가져와 우리 서비스에 다시 보여주면" 민사(②④⑤)는 그대로 살아남.**
특히 **MCP 서버는 서버사이드**라 "브라우저 확장(클라이언트 분석)" 같은 안전한 탈출구를 못 씀 = 우리 서버가 직접 긁는 구조 = 리스크 최대.

**안전 구간**: 원문 비저장 + 부분 인용 + 출처 표시 + 링크아웃 + 실시간 조회.

## 1-3. 합법 수집 가능 데이터 지도

### ❌ 못 가져오는 것 (확인 완료)
- 쿠팡·네이버쇼핑·11번가·G마켓 **별점/리뷰 원문·평점·리뷰수 → 전부 미제공**
- 다나와 OpenAPI 종료 / 글로우픽·화해 공개 API 없음
- 레딧($1.2만/년+승인) · X(전면 유료) · 인스타(승인+7일한도) → 사실상 막힘
- 국내 커뮤니티(디시·클리앙·뽐뿌) → 공식 API/RSS 자체가 없음
- 구글 Custom Search → 신규 가입 차단(2027 종료)
- 공공데이터 → 개별 상품 리뷰 데이터셋 없음
- 해외만 조건부: 아마존(별점·리뷰수, 본문❌, 승인필요, 2026.5 폐기) / 알리(평균평점)

### ✅ 합법적으로 가져올 수 있는 것

| 소스 | 주는 데이터 | 쿼터 | 핵심 제약 |
|---|---|---|---|
| **네이버 검색 API** | 블로그·카페·뉴스 글 제목+**요약 스니펫**(전문❌)+작성자+날짜+링크 | 2.5만회/일 | 장기 DB적재·결과옆 광고·제3자 재공급 ❌ |
| **네이버 쇼핑 API** | 가격·최저가·판매몰·카테고리·상품ID | 2.5만회/일 | 평점·리뷰 필드 없음 |
| **네이버 DataLab** | 검색어 트렌드·인구통계(상대값) | 1천회/일 | 카테고리/키워드 단위 |
| **YouTube Data API** | 리뷰영상 검색·조회수·좋아요·댓글 | 1만 units/일 | 자막은 본인영상만, 30일 캐시규칙 |
| **Gemini(영상이해)** | 유튜브 URL 직접 "보고" 장단점 요약 | 별도(미확인) | 공개영상·일일 영상시간 한도 |
| **쿠팡 파트너스** | 상품·최저가·딥링크(수익화) | 시간당 ~10회 | 리뷰·평점 ❌ |

> **유튜브 보충**: ① 텍스트 메타+댓글(공식 API, 합법) ② 자막(남의 영상은 합법적으로 막힘) ③ **Gemini가 영상 URL 직접 보고 요약**(합법, 가장 현실적). 진짜 영상 내용을 쓰려면 ③.

# PART 3. 종합 인사이트 & 유력 방향

## 3-1. 핵심 인사이트 3가지
1. **"가짜리뷰 거르는 신뢰도"는 PlayMCP의 검증된 승리 패턴**(신뢰·안전 클러스터가 크고 인기). 우리 방향성은 옳다.
2. **단, 맛집은 이미 2팀 점령. 상품(쇼핑)은 비어있음** → 빈자리 = **"찐맛집의 쇼핑 버전"**
3. **재료가 공개됨 = 양날의 검.** 네이버검색·유튜브 MCP 오픈소스라 개발 빠름(✅)이자 진입장벽 낮음(⚠️) → 차별점은 "조합"이 아니라 **신뢰도 알고리즘 + UX 완성도**

## 3-2. 유력 방향 (가설)
**"광고·협찬 거른 진짜 상품 후기 큐레이션 + 신뢰도 점수" MCP**

```
카톡: "에어팟 프로 살까?"
  → ① 네이버 검색 API: 블로그·카페 실사용 후기 N개 (스니펫+링크)
  → ② YouTube + Gemini: 리뷰 영상 보고 장단점 요약
  → ③ 네이버쇼핑/쿠팡파트너스: 가격대 + 최저가 바로가기
  → ④ DataLab: 관심도 트렌드 신호
  → ⑤ LLM 종합: "장점 / 단점 / 이런 사람에게 / 어디서 사면 이득 + 출처"
     + 광고·협찬 문구 자동 감지해 제외, 신뢰도 점수화
```
- 별점 원문 없이 "진짜 장문 후기"만 → 약점이 오히려 컨셉(못 믿을 데이터를 빼고 진짜만)
- 원문 비저장 + 스니펫 + 링크아웃 → 약관·저작권 안전, 데모 안 막힘(안정성 ✅)

## 3-3. 솔직한 종합 판단
- **기회**: 합법 형태 존재, 2주 내 구현 가능, 신뢰·안전 패턴은 검증됨, 쇼핑 버전은 빈자리
- **리스크**: 원래 그림보다 좁아짐 / 저관여 상품은 블로그 후기 자체가 없어 커버리지 구멍 / 컨셉 자체는 해외 유사품 존재 → 창의성 점수는 데이터가 아니라 **UX·완성도**에서 승부
